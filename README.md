Телеграм-бот для школьного класса на **Python + aiogram 3**. Класс один раз настраивает расписание уроков и время звонков, а дальше бот сам напоминает о домашних заданиях и уроках на завтра — прямо в чат.

---

## ✨ Возможности

| | |
|---|---|
| 🚀 **Онбординг** | Пошаговая настройка: количество уроков, время звонков, расписание на каждый день недели (с опциональной субботой). Повторная настройка уже настроенного чата требует явного подтверждения и сохраняется одной атомарной транзакцией |
| 📚 **Сегодня** | Сводка на день: уроки сегодня, домашка к сдаче, просроченные задания |
| 📅 **Расписание** | Просмотр расписания по дням недели с указанием времени уроков |
| 📝 **Домашние задания** | Добавление, редактирование и отметка о выполнении домашки по предметам и датам, архив выполненного, подтверждение перед безвозвратным удалением |
| ⏰ **Напоминания** | Ежедневная рассылка по расписанию: домашка на завтра + просроченные задания + уроки на завтра. Доставка идемпотентна и переживает перезапуск бота |
| ⚙️ **Настройки** | Индивидуальное время напоминаний для чата, включение/отключение каждого типа напоминаний, полный сброс настроек |
| 💬 **Личка и группы** | Работает как в личном чате, так и в общем чате класса — настройки хранятся отдельно на каждый `chat_id`, с разграничением прав (см. ниже) |

Под капотом — конечный автомат (FSM) для диалогов с персистентным хранилищем состояния, автоматическое разбиение длинных сообщений на части (лимит Telegram в 4096 символов), HTML-форматирование с экранированием пользовательского ввода и планировщик задач на APScheduler с учётом часового пояса и outbox-таблицей для надёжной доставки.

## 🔐 Права доступа

- **В личном чате** ограничений нет — единственный пользователь может делать всё.
- **В группе/супергруппе** только администраторы чата могут:
  - полностью сбрасывать данные (⚙️ Настройки → «Сбросить всё»);
  - менять расписание и время звонков (первичная настройка, переконфигурация, редактирование расписания);
  - менять настройки и время напоминаний.
- **Домашние задания** (добавление, редактирование, отметка выполнения, удаление) доступны **любому участнику** группы — это совместный список класса. Удаление ДЗ дополнительно требует подтверждения («Удалить безвозвратно?»), чтобы случайный тап не уничтожал запись без права на отмену.

При отказе бот показывает понятное сообщение («🚫 Это действие доступно только администраторам чата.») вместо тихого игнорирования действия.

## 🏗️ Архитектура

```
telegram-school-bot/
├── bot.py             # Точка входа: миграции, middleware, роутеры, планировщик
├── config.py          # Конфигурация из переменных окружения + валидация
├── utils.py           # Разбиение сообщений, HTML-экранирование, safe-парсинг callback_data и др.
├── handlers/          # Обработчики диалогов и команд
│   ├── onboarding.py  #   первичная настройка/переконфигурация расписания
│   ├── today.py        #   сводка на сегодня
│   ├── schedule.py    #   просмотр и редактирование расписания
│   ├── homework.py    #   CRUD домашних заданий
│   ├── settings.py    #   настройки напоминаний, полный сброс
│   ├── migration.py   #   миграция group → supergroup
│   └── common.py      #   /help, /cancel, /start
├── keyboards/         # Reply/inline клавиатуры
├── middleware/
│   └── access.py      #   контекст чата, guard онбординга, проверка прав админа
├── database/          # SQLAlchemy-модели, async-доступ к БД, FSM-хранилище
│   ├── models.py      #   модели + constraints/индексы
│   ├── db.py           #   CRUD, атомарный онбординг, outbox-джобы
│   ├── fsm_storage.py  #   персистентное FSM-хранилище на SQLite
│   └── migrate.py      #   запуск Alembic-миграций при старте
├── alembic/            # Версионируемые миграции схемы БД
├── services/
│   └── scheduler.py   #   ежедневные напоминания через APScheduler + outbox
└── tests/              # pytest + pytest-asyncio: БД, хендлеры, планировщик, права доступа
```

**Стек:** aiogram 3 (async), SQLAlchemy 2 + aiosqlite (async ORM над SQLite), Alembic (миграции), APScheduler, pytz, python-dotenv.

**Поддерживаемая БД:** только SQLite (`DATABASE_URL` обязан начинаться с `sqlite`) — это осознанное ограничение текущей версии, зафиксированное в `config.py`.

## ⚙️ Установка и запуск

### Локально

```bash
git clone https://github.com/blikzes-bit/telegram-school-bot.git
cd telegram-school-bot

python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

pip install -r requirements-dev.txt   # прод + тесты/линт/типы

cp .env.example .env          # укажи свой BOT_TOKEN

python bot.py
```

При старте `bot.py` сам прогоняет `alembic upgrade head`, поднимая схему БД до актуальной версии (см. раздел «Миграции БД» ниже).

### Docker

Готовый мультиархитектурный образ (`amd64` + `arm64`) публикуется в GHCR при каждом пуше в `main`. Продакшн-образ собирается в два этапа (builder/final), не содержит тестов и dev-инструментов, запускается от непривилегированного пользователя и имеет встроенный `HEALTHCHECK`:

```bash
docker run -d \
  --name school-bot \
  -e BOT_TOKEN=your_token_here \
  -e TIMEZONE=Europe/Kiev \
  -v school_bot_data:/data \
  ghcr.io/blikzes-bit/telegram-school-bot:latest
```

База данных хранится в `/data/school_bot.db` — том нужен для сохранения данных между перезапусками контейнера.

### Переменные окружения

| Переменная | Обязательна | По умолчанию | Описание |
|---|---|---|---|
| `BOT_TOKEN` | ✅ | — | Токен бота от [@BotFather](https://t.me/BotFather) |
| `TIMEZONE` | ❌ | `Europe/Kiev` | Часовой пояс для расчёта времени напоминаний |
| `DATABASE_URL` | ❌ | `sqlite+aiosqlite:///school_bot.db` | Строка подключения к БД (только SQLite) |
| `FSM_STORAGE` | ❌ | `sqlite` | `sqlite` — персистентное хранилище состояний диалогов (переживает перезапуск), `memory` — только для локальной разработки |
| `HEARTBEAT_FILE` | ❌ | `.heartbeat` | Файл, который планировщик обновляет каждую минуту; используется Docker `HEALTHCHECK` |

## 🗃️ Миграции БД

Схема версионируется через **Alembic** (`alembic/`) вместо ручных `ALTER TABLE`. При обычном запуске `bot.py` сам вызывает `alembic upgrade head` — никаких дополнительных действий не требуется для новой установки.

Для **существующей** продакшн-базы, созданной до внедрения Alembic (т.е. любой `school_bot.db`, работавший со старой версией бота), перед первым запуском новой версии один раз выполните:

```bash
alembic stamp e42f61dd6f2f   # baseline-ревизия, соответствующая старой схеме
alembic upgrade head          # применит constraints/индексы/outbox/FSM-таблицы
```

### Миграция group → supergroup

Когда Telegram обновляет обычную группу до супергруппы, chat_id меняется. Бот отслеживает системное сообщение об этом и переносит все данные (расписание, звонки, домашние задания, настройки) на новый chat_id в одной транзакции — вручную ничего переносить не нужно.

## 🧪 Тесты и качество кода

```bash
pytest                                  # полный набор тестов
pytest --cov=. --cov-report=term-missing  # с покрытием
ruff check .                            # линт
mypy .                                  # проверка типов
pip-audit -r requirements.txt           # аудит зависимостей на уязвимости
```

Тесты покрывают: работу с БД и конкурентный `get_or_create_chat`, атомарность онбординга и переконфигурации, поведение до `/start` и после полного сброса, права доступа участник/админ в группе, безопасный HTML-рендеринг, устаревшие/битые callback-кнопки, edge-cases дат (29 февраля, просроченные быстрые кнопки), outbox-доставку напоминаний (частичный сбой, `RetryAfter`, `Forbidden`) и миграцию group → supergroup. GitHub Actions прогоняет полный CI (лint + типы + тесты + аудит) на каждый pull request, а тесты — перед публикацией Docker-образа.

## ⚠️ Известные ограничения

- Только SQLite — другие БД не поддерживаются и явно отклоняются при старте.
- Троттлинг отправки напоминаний — простая пауза между сообщениями, не полноценный token-bucket.
- Часть межчатовых запросов планировщика (по чатам целиком) не батчится в один SQL-запрос — при очень большом числе чатов возможен рост числа запросов к БД.
